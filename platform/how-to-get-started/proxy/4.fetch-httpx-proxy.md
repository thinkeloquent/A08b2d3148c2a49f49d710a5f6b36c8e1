## Proxy Class

**File:** `_config.py` (lines 184-233)

The `Proxy` dataclass wraps a proxy URL with optional authentication and headers:

```python
from fetch_httpx._config import Proxy

# From URL string
proxy = Proxy("http://user:pass@proxy.example.com:8080")

# With explicit auth and headers
proxy = Proxy(
    "http://proxy.example.com:8080",
    auth=("username", "password"),
    headers={"Proxy-Authorization": "Bearer token"},
)
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `url` | `URL` | Proxy URL (converted from string) |
| `auth` | `tuple[str, str] \| None` | Username/password tuple |
| `headers` | `dict[str, str] \| None` | Additional proxy headers |

## Type Definitions

**File:** `_types.py` (lines 129-136)

```python
# Single proxy: URL string, Proxy object, or None
ProxyTypes = str | Proxy | None

# Multiple proxies: single proxy or scheme-keyed dict
ProxiesTypes = ProxyTypes | dict[str, ProxyTypes]
```

The dict form allows per-scheme routing:

```python
proxies = {
    "https://": "http://https-proxy:8080",
    "http://": "http://http-proxy:8080",
}
```

### Proxy Resolution (`_resolve_proxy`)

The `_resolve_proxy()` method normalizes `ProxiesTypes` to a single `Proxy`:

```
Input                             Output
─────────────────────────────     ──────────────────────────
None                              None
"http://proxy:8080"               Proxy("http://proxy:8080")
Proxy(...)                        Proxy(...) (pass-through)
{"https://": "http://p:8080"}     Proxy("http://p:8080")  (prefer https:// key)
{"all://": "http://p:8080"}       Proxy("http://p:8080")  (then all://)
{"http://": "http://p:8080"}      Proxy("http://p:8080")  (then first non-None)
```

### Transport Wiring (`_ensure_transport`)

The resolved `Proxy` is passed to `AsyncHTTPTransport`:

```python
async def _ensure_transport(self) -> AsyncHTTPTransport:
    if self._transport is None:
        proxy = self._resolve_proxy()
        self._transport = AsyncHTTPTransport(
            verify=self._verify,
            cert=self._cert,
            http1=self._http1,
            http2=self._http2,
            limits=self._limits,
            timeout=self._timeout,
            proxy=proxy,          # <-- passed to transport
        )
    return self._transport
```

## Transport Layer

**File:** `_transports/_http.py`

### AsyncHTTPTransport (lines 224-296)

The transport accepts a `Proxy` object and converts it to httpx's format:

```python
class AsyncHTTPTransport:
    def __init__(self, *, proxy: Proxy | None = None, ...):
        self._proxy = proxy

    async def _ensure_transport(self):
        self._transport = httpx.AsyncHTTPTransport(
            proxy=str(self._proxy.url) if self._proxy else None,
            # ...other options
        )
```

httpx handles the actual CONNECT tunnel through the proxy for HTTPS requests.

### Credential Masking (lines 26-39)

`_mask_proxy_url()` redacts embedded credentials for log output:

```
http://user:pass@proxy:8080  ->  http://***:***@proxy:8080
```

## Sync Client

The `Client` (sync) class delegates to `AsyncClient` internally, so all proxy
configuration applies identically:

```python
from fetch_httpx import Client

with Client(proxies="http://proxy:8080") as client:
    response = client.get("https://api.example.com/health")
```

## fetch_httpx_clients Factory

**File:** `packages_py/fetch_httpx_clients/fetch_httpx_clients/clients.py`

The `fetch_httpx_async()` and `fetch_httpx_sync()` factory functions are
LLM-tuned wrappers. They support proxy via `trust_env`:

```python
from fetch_httpx_clients import fetch_httpx_async

client = fetch_httpx_async(
    base_url="https://api.openai.com/v1",
    headers={"Authorization": "Bearer sk-..."},
    trust_env=True,  # Reads HTTP_PROXY / HTTPS_PROXY
)
```

To pass an explicit proxy URL, create `AsyncClient` directly instead of using
the factory.
