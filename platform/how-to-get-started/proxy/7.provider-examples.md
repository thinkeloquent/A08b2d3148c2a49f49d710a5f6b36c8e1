## Adding Proxy to a Provider

### 1. Direct URL in YAML

```yaml
providers:
  saucelabs:
    base_url: "https://api.us-west-1.saucelabs.com/rest/v1"
    proxy_url: "http://proxy.corp.local:8080"
```

### 2. Authenticated Proxy in YAML

```yaml
providers:
  rally:
    base_url: "https://rally1.rallydev.com/slm/webservice/v2.0"
    proxy_url: "http://user:p%40ssword@proxy.corp.local:8080"
```

Note: credentials must be percent-encoded. Use `proxy_url_builder` to construct
the URL from raw credentials safely.

### 3. Environment Variable via Overwrite

```yaml
providers:
  figma:
    proxy_url: false              # Default: disabled
    overwrite:
      proxy_url: "{{env.FIGMA_PROXY_URL}}"   # Override when env var is set
```

When `FIGMA_PROXY_URL` is set in the environment, the overwrite system replaces
`proxy_url: false` with the resolved URL. When unset, the overwrite resolves to
`null` which means "inherit global" — be careful this doesn't accidentally
enable proxying.

### 4. Inherit Global Proxy

```yaml
providers:
  servicenow:
    base_url: "https://instance.service-now.com"
    proxy_url: null   # Inherits global.network.proxy_urls[environment]

global:
  network:
    proxy_urls:
      dev: "http://proxy.dev.local:8080"
      prod: "http://proxy.prod.local:8080"
```

## Switching Environments (Cloud Deployment)

In cloud environments where direct connections are blocked, update the provider
config or use the overwrite system:

```yaml
# server.prod.yaml (or environment-specific overwrite)
providers:
  figma:
    proxy_url: "http://egress-proxy.internal:3128"
  saucelabs:
    proxy_url: "http://egress-proxy.internal:3128"
  rally:
    proxy_url: "http://egress-proxy.internal:3128"
  statsig:
    proxy_url: "http://egress-proxy.internal:3128"
```

Or globally:

```yaml
global:
  network:
    proxy_urls:
      prod: "http://egress-proxy.internal:3128"

providers:
  figma:
    proxy_url: null    # null = inherit global
  saucelabs:
    proxy_url: null
```

## Runtime Override (Ad-Hoc Testing)

Use the POST endpoint to test a different proxy without restarting:

```bash
# Test figma with a specific proxy
curl -X POST http://localhost:51000/healthz/admin/integration/figma \
  -H "Content-Type: application/json" \
  -d '{"proxy_url": "http://test-proxy:8080"}'

# Test direct connection (disable proxy)
curl -X POST http://localhost:51000/healthz/admin/integration/rally \
  -H "Content-Type: application/json" \
  -d '{"proxy_url": false}'
```
