# fetch-undici Proxy Implementation (Node.js)

Source: `platform/packages_mjs/fetch_undici/src/`

## Proxy Class

**File:** `config/proxy.ts`

The `Proxy` class wraps a proxy URL with authentication, headers, and no-proxy
bypass logic:

```typescript
import { Proxy, createProxy, getEnvProxy } from "fetch-undici";

// From a URL string
const proxy = new Proxy("http://user:pass@proxy.example.com:8080");

// From options object
const proxy = new Proxy({
  url: "http://proxy.example.com:8080",
  auth: { username: "user", password: "pass" },
  headers: { "Proxy-Authorization": "Bearer token" },
  noProxy: ["*.internal.com", "localhost"],
});

// From various input types (string, ProxyOptions, Proxy, null)
const proxy = createProxy(input); // returns Proxy | null
```

### Key Methods

| Method              | Description                                                                          |
| ------------------- | ------------------------------------------------------------------------------------ |
| `toUndiciOptions()` | Converts to `{ uri, token? }` for `new ProxyAgent(...)`                              |
| `shouldBypass(url)` | Checks URL against `noProxy` patterns (exact, wildcard `*.domain`, suffix `.domain`) |
| `sanitizedUrl`      | Returns URL without credentials (safe for logging)                                   |

### Credential Extraction

If the proxy URL contains embedded credentials (`http://user:pass@host:port`),
they are automatically extracted and stored in `auth`. The `toUndiciOptions()`
method converts them to a Base64 `Basic` token for `ProxyAgent`:

```typescript
// proxy.toUndiciOptions() returns:
{
  uri: "http://user:pass@proxy.example.com:8080",
  token: "Basic ***REDACTED***"  // Base64("user:pass")
}
```

### Environment Variables

`getEnvProxy()` reads standard proxy env vars:

```typescript
const envProxy = getEnvProxy();
// envProxy.http   -> Proxy from HTTP_PROXY / http_proxy
// envProxy.https  -> Proxy from HTTPS_PROXY / https_proxy (falls back to HTTP_PROXY)
// envProxy.noProxy -> string[] from NO_PROXY / no_proxy (comma-separated)
```

## AsyncClient Integration

**File:** `client/client.ts` (constructor, lines 154-176)

When creating an `AsyncClient`, proxy support is activated in two ways:

### Explicit Proxy (AppYamlConfig flow)

```typescript
const client = new AsyncClient({
  baseUrl: "https://api.figma.com/v1",
  proxy: "http://proxy.example.com:8080", // string, ProxyOptions, or Proxy
  // trust_env defaults to false when proxy is provided
});
```

Internally:

1. `createProxy(opts.proxy)` normalizes input to a `Proxy` instance
2. `new ProxyAgent(proxy.toUndiciOptions())` creates the undici proxy dispatcher
3. The `ProxyAgent` is mounted on the `MountRouter` for both `https://` and `http://`
4. All requests matching those patterns route through the proxy

### Environment Trust (fallback)

```typescript
const client = new AsyncClient({
  baseUrl: "https://api.figma.com/v1",
  trustEnv: true, // or trust_env (snake_case alias)
});
```

When `trustEnv` is true and no explicit proxy is set, the constructor calls
`getEnvProxy()` and mounts separate `ProxyAgent` instances for HTTP and HTTPS.

### Options

```typescript
interface AsyncClientOptions {
  proxy?: Proxy | ProxyOptions | string | null;
  trustEnv?: boolean; // Honor HTTP_PROXY / HTTPS_PROXY env vars
  trust_env?: boolean; // Snake-case alias
}
```

## MountRouter Dispatch

**File:** `transport/router.ts`

The `MountRouter` routes requests to dispatchers by URL pattern. Proxy agents
are mounted on scheme-level patterns:

```
Pattern          Dispatcher        Specificity
─────────────    ───────────────   ───────────
https://         ProxyAgent        10
http://          ProxyAgent        10
all://           (default pool)     0
```

When `_getDispatcher(url)` is called:

1. Check `MountRouter` for matching pattern (most specific first)
2. If a proxy agent is mounted on `https://`, all HTTPS requests go through it
3. If no match, fall back to `DispatcherFactory` (direct connection pool)

## Client Options Definition

**File:** `client/options.ts`

`normalizeClientOptions()` handles alias normalization:

```typescript
proxy: options.proxy,                              // passed through
trustEnv: options.trustEnv ?? options.trust_env,    // camelCase preferred
```

## Lifecycle

On `client.close()`, the `MountRouter.closeAll()` method closes all mounted
dispatchers including any proxy agents, releasing their connection pools.
