# Proxy Debugging Guide

## Common Symptoms

### "Connection timed out" with proxy configured

**Cause:** Proxy is accepted in config but not wired to the transport layer.

**Check:** Look for these log lines:

```
# Node.js (fetch-undici)
[WARN] No dispatcher found for URL    <- proxy agent not mounted
[INFO] Created new dispatcher          <- direct connection (bypasses proxy)
[ERROR] Connect Timeout Error          <- times out because direct is blocked

# Python (fetch-httpx)
[ERROR] All connection attempts failed <- proxy not passed to transport
```

**Fix:** Ensure the `proxy` option is being passed to `AsyncClient` and that
the client implementation mounts it (see architecture docs).

### "Connection refused" to proxy

**Cause:** Proxy URL is wrong, proxy is down, or credentials are rejected.

**Check:** Test with curl first:

```bash
curl -x "http://proxy:8080" https://api.figma.com/v1/me \
  -H "X-Figma-Token: your-token"
```

### Proxy works with curl but not the app

**Cause:** The app's HTTP client is not routing through the proxy. This was a
known bug where both `fetch-undici` and `fetch-httpx` accepted proxy config
but did not wire it to the transport.

**Resolution:**
- **fetch-undici:** Fixed by mounting `ProxyAgent` on `MountRouter` in `AsyncClient` constructor
- **fetch-httpx:** Fixed by passing `proxy=` to `AsyncHTTPTransport` in `_ensure_transport()`

## Diagnostic Endpoints

### Health Check with Proxy Details

```bash
# GET — shows resolved proxy config and connection result
curl http://localhost:51000/healthz/admin/integration/figma | jq .

# Response includes:
{
  "healthy": true,
  "connection_details": {
    "proxy": {
      "configured": true,
      "mode": "explicit",           // "explicit" | "system_env" | "disabled"
      "url": "http://proxy:8080"
    }
  }
}
```

### Config Debug

```bash
# Show full resolved config (credentials masked)
curl http://localhost:51000/healthz/admin/integration/figma/config | jq .
```

### Override Proxy at Runtime

```bash
# Test with different proxy
curl -X POST http://localhost:51000/healthz/admin/integration/figma \
  -H "Content-Type: application/json" \
  -d '{"proxy_url": "http://alt-proxy:8080"}'
```

## Log Lines to Watch

### fetch-undici (Node.js)

```
[DEBUG] Proxy created                         <- Proxy class instantiated
[INFO]  Proxy mounted on router               <- ProxyAgent registered on MountRouter
[DEBUG] Matched mount pattern https://        <- request routed through proxy
[WARN]  No dispatcher found for URL           <- proxy NOT mounted (bug)
[DEBUG] Created new dispatcher                <- direct connection created
[ERROR] Connect Timeout Error                 <- direct connection failed
```

### fetch-httpx (Python)

```
[WARN]  Proxy configured                      <- proxy stored on client
[WARN]  Proxy configured for async transport  <- proxy passed to transport
[INFO]  AsyncHTTPTransport created            <- transport with proxy ready
[DEBUG] Sending async request                 <- request via transport
[ERROR] All connection attempts failed        <- proxy not routing (bug)
```

### Health Diagnostics

```
[fetch_config] {
  provider: "figma",
  proxy_url: "http://proxy:8080",     <- resolved from AppYamlConfig
  ...
}
```

## Environment Variable Checklist

| Variable | Used By | Notes |
|----------|---------|-------|
| `HTTPS_PROXY` | Both | Used when `trust_env=true` and no explicit proxy |
| `HTTP_PROXY` | Both | Fallback for HTTPS if HTTPS_PROXY not set |
| `NO_PROXY` | Both | Comma-separated list of bypass patterns |
| `FIGMA_PROXY_URL` | Figma env-resolver | Provider-specific override |

## Verifying Proxy is Active

### Node.js

After creating an `AsyncClient` with proxy, check the logs for:

```
[INFO] Proxy mounted on router { url: "http://proxy:8080" }
```

If you see `No dispatcher found for URL` instead, the proxy was not mounted.

### Python

After creating an `AsyncClient` with proxy, check for:

```
[WARN] Proxy configured for async transport { proxy: "http://***:***@proxy:8080" }
```

If you only see `AsyncHTTPTransport created` without the proxy line, the proxy
was not passed to the transport.
