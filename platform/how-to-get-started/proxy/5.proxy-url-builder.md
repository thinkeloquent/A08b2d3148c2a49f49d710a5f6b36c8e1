# proxy_url_builder — Authenticated Proxy URL Construction

Source: `platform/polyglot/proxy_url_builder/`

A polyglot utility (TypeScript + Python) that builds proxy URLs with
percent-encoded credentials from separate parts. Cross-language output parity
is tested.

## TypeScript

**File:** `mjs/src/index.ts`

```typescript
import { buildProxyUrl, encodeCredential } from "proxy_url_builder";

// Build authenticated proxy URL
const url = buildProxyUrl("user@domain", "p@ss:word!", "proxy.example.com:8080");
// -> "http://user%40domain:p%40ss%3Aword%21@proxy.example.com:8080"

// Input can be:
//   bare hostname:      "proxy.example.com"
//   hostname:port:      "proxy.example.com:8080"
//   full URL:           "http://proxy.example.com:8080"
//   URL with old creds: "http://old:creds@proxy.example.com:8080" (replaced)

// Encode a single credential value
const encoded = encodeCredential("user@domain");
// -> "user%40domain"
```

### Normalization Rules

1. If input has no `://`, prepends `http://`
2. Existing credentials in the URL are replaced
3. Default ports are stripped (80 for http, 443 for https)
4. Trailing paths are stripped
5. Credentials are percent-encoded per RFC 3986 userinfo rules

## Python

**File:** `py/proxy_url_builder/builder.py`

```python
from proxy_url_builder import build_proxy_url, encode_credential

# Build authenticated proxy URL
url = build_proxy_url("user@domain", "p@ss:word!", "proxy.example.com:8080")
# -> "http://user%40domain:p%40ss%3Aword%21@proxy.example.com:8080"

# Encode a single credential value
encoded = encode_credential("user@domain")
# -> "user%40domain"
```

Uses `urllib.parse.quote(value, safe='')` which matches the TypeScript
`encodeURIComponent` + additional RFC 3986 encoding.

## When to Use

Use `proxy_url_builder` when constructing proxy URLs from separate credential
fields (e.g. from vault secrets or environment variables):

```yaml
# AppYamlConfig overwrite pattern
figma:
  overwrite:
    proxy_url: "{{fn:build_proxy_url(env.PROXY_USER, env.PROXY_PASS, env.PROXY_HOST)}}"
```

Do NOT use it when:
- The proxy URL already contains credentials
- No authentication is needed (just use the URL string directly)
- The proxy uses header-based auth (use `headers` on the Proxy object instead)
