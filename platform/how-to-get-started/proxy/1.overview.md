# Proxy Implementation Overview

The platform provides a polyglot proxy system that routes HTTP requests through
forward proxies. Both the Node.js (`fetch-undici`) and Python (`fetch-httpx`)
HTTP clients share the same configuration semantics and resolution order.

## Architecture

```
AppYamlConfig (server.dev.yaml)
  providers.<name>.proxy_url           per-provider proxy
  global.network.proxy_urls            environment-keyed global proxies
  global.network.agent_proxy           HTTP_PROXY / HTTPS_PROXY equivalents
        |
        v
  applyResolvedOverwrites()            runtime compute / overwrites
        |
        v
  healthz_integration route            resolves final proxy URL
        |
        +--- fetch-undici AsyncClient   Node.js path
        |      proxy -> ProxyAgent -> MountRouter
        |
        +--- fetch-httpx AsyncClient    Python path
               proxies -> Proxy -> AsyncHTTPTransport -> httpx
```

## Key Packages

| Package | Language | Role |
|---------|----------|------|
| `fetch-undici` | TypeScript | HTTP client — proxy via `ProxyAgent` + `MountRouter` |
| `fetch-httpx` | Python | HTTP client — proxy via `Proxy` + `httpx` transport |
| `proxy_url_builder` | Both | Builds authenticated proxy URLs from credentials |
| `app_yaml_overwrites` | Both | Runtime overwrite layer for `proxy_url` |
| `env-resolver` | TypeScript | Per-provider env var resolution (e.g. `FIGMA_PROXY_URL`) |

## proxy_url Semantics

The `proxy_url` field on each provider has three possible states:

| Value | Meaning |
|-------|---------|
| `false` | Explicitly disabled. Forces direct connection, ignores global proxy and env vars. |
| `null` / missing | No override. Inherits global proxy (`global.network.proxy_urls`) or env vars. |
| `"http://..."` | Explicit proxy URL. Used as-is. Supports embedded credentials. |

These semantics are documented in `server.dev.yaml` lines 70-71 and enforced by
the health check routes in both Fastify and FastAPI.
