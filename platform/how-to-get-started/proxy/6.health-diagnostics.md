# Health Diagnostics — Proxy-Aware Client Creation

The health check routes create proxy-aware HTTP clients for every provider.
Both the Fastify (Node.js) and FastAPI (Python) routes follow the same pattern.

## Proxy Resolution Flow

```
                    AppYamlConfig
                         |
              applyResolvedOverwrites()
                         |
                   providerConfig
                         |
               providerConfig.proxy_url
                    /    |    \
                   /     |     \
              false    string    null
                |        |        |
           No proxy  Use URL  Env fallback
           trust=F   trust=F   trust=T
```

## Fastify Route (Node.js)

**File:** `fastify_server/routes/healthz_integration.route.mjs`

### Standard Client Creation (lines 447-506)

```javascript
// Resolve proxy from provider config
const proxyUrlConfig = providerConfig.proxy_url;
const envProxy = getEnvProxy();

const resolvedProxy = proxyUrlConfig === false
  ? null                                                    // Explicitly disabled
  : (typeof proxyUrlConfig === "string" && proxyUrlConfig
    ? proxyUrlConfig                                        // Explicit URL
    : (envProxy.https || envProxy.http || null));            // Env fallback

const client = new AsyncClient({
  baseUrl,
  headers: { ...customHeaders, ...authHeaders },
  timeout: timeoutMs,
  verify: providerConfig.verify_ssl !== false,
  proxy: resolvedProxy,
  trust_env: proxyUrlConfig == null,  // Only trust env when proxy_url not set
});
```

### Connection Details Logging (lines 260-296)

The health response includes proxy diagnostics:

```javascript
let proxyDetails;
if (proxyUrlConfig === false) {
  proxyDetails = { configured: false, mode: "disabled", url: null };
} else if (typeof proxyUrlConfig === "string" && proxyUrlConfig) {
  proxyDetails = { configured: true, mode: "explicit", url: proxyUrlConfig };
} else if (proxyUrlConfig === null || proxyUrlConfig === undefined) {
  proxyDetails = {
    configured: true, mode: "system_env",
    HTTPS_PROXY: process.env.HTTPS_PROXY,
    HTTP_PROXY: process.env.HTTP_PROXY,
    NO_PROXY: process.env.NO_PROXY,
  };
}
```

### Runtime Override via POST (lines 696-711)

The POST endpoint accepts proxy overrides for ad-hoc testing:

```
POST /healthz/admin/integration/<provider>
Content-Type: application/json

{
  "proxy_url": "http://alternate-proxy:8080",
  "verify_ssl": false,
  "ca_bundle": "/path/to/ca.pem"
}
```

The `_applyOverrides()` function merges these into the provider config before
client creation. The response includes `_overrides_applied.proxy: true` to
confirm the override was used.

## FastAPI Route (Python)

**File:** `fastapi_server/routes/healthz_integration.route.py`

### Standard Client Creation (lines 436-454)

```python
proxy_url_config = provider_config.get("proxy_url")

if proxy_url_config is False:
    proxies = None
    trust_env = False
elif isinstance(proxy_url_config, str) and proxy_url_config:
    proxies = proxy_url_config
    trust_env = False
else:
    proxies = None
    trust_env = True

return AsyncClient(
    base_url=base_url,
    headers=all_headers,
    timeout=timeout_ms / 1000,
    verify=provider_config.get("verify_ssl", True),
    proxies=proxies,
    trust_env=trust_env,
)
```

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/healthz/admin/integration/<provider>` | Health check with resolved proxy config |
| POST | `/healthz/admin/integration/<provider>` | Health check with runtime proxy override |
| GET | `/healthz/admin/integration/<provider>/config` | Show resolved config (proxy masked) |

## Integration Registry

Each provider entry in the integration registry (`INTEGRATIONS` object) declares
which environment variables are relevant for diagnostics:

```javascript
{
  figma: {
    envVars: [
      "FIGMA_TOKEN", "FIGMA_ACCESS_TOKEN",
      "FIGMA_PROXY_URL",
      "HTTPS_PROXY", "HTTP_PROXY", "NO_PROXY",
    ],
    // ...
  },
}
```

These are included in the health response for troubleshooting proxy issues.
